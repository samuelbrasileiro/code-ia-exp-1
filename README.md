# Exam Builder

Create multiple-choice exams, generate randomized PDF variants, export answer keys, and grade student submissions from CSV. The app is a simple monorepo with a React UI and an Express API.

**Features**
- Manage questions with multiple correct choices and optional tags
- Build exams by selecting questions and choosing an answer labeling mode
- Generate randomized exam variants and PDFs
- Export answer keys as CSV (letters or powers-of-two mode)
- Grade student answers from CSV with strict or lenient scoring
- Download a ZIP with multiple PDF copies plus the answer key

**Tech Stack**
- Frontend: React 18, Vite, React Router
- Backend: Node.js, Express, Zod validation
- PDF/ZIP: PDFKit, Archiver
- CSV parsing: csv-parse
- Tests: Cucumber (acceptance tests), Supertest
- Storage: JSON files on disk (`data/` by default)

**Project Structure**
- `apps/frontend` - React UI
- `apps/backend` - Express API and file-backed storage
- `packages/shared` - Shared TypeScript types
- `data` - JSON persistence (auto-created, gitignored)

Run Backend (dev)

**API Reference**
Base URL: `http://localhost:3000`

Health
- `GET /health` -> `{ "status": "ok" }`

Questions
- `GET /api/questions`
- `POST /api/questions`
- `PUT /api/questions/:id`
- `DELETE /api/questions/:id`

Question payload
```json
{
  "prompt": "What is 2 + 2?",
  "choices": ["3", "4", "5"],
  "correctIndexes": [1],
  "tags": ["math", "intro"]
}
```

Notes
- `correctIndexes` are zero-based indexes into `choices`.
- Multiple correct answers are allowed.

Exams
- `GET /api/exams`
- `GET /api/exams/:id`
- `POST /api/exams`
- `PUT /api/exams/:id`
- `DELETE /api/exams/:id`
- `POST /api/exams/:id/variant`
- `POST /api/exams/:id/pdf`
- `POST /api/exams/:id/pdf-zip`
- `GET /api/exams/:id/pdf-history`
- `POST /api/exams/:id/answer-key`

Exam payload
```json
{
  "title": "Midterm Exam",
  "subject": "Mathematics",
  "teacher": "Prof. Ana",
  "date": "2026-03-26",
  "answerLabelingMode": "letters",
  "questionIds": ["q1", "q2", "q3"]
}
```

Variant response (from `POST /api/exams/:id/variant`)
```json
{
  "examId": "...",
  "variantId": "...",
  "examNumber": "A001",
  "questions": [
    { "questionId": "...", "shuffledChoiceIds": ["...", "..."] }
  ]
}
```

Generate a single PDF
- `POST /api/exams/:id/pdf`
- Body: `{ "variant": <ExamVariant> }`
- Response: `application/pdf`

Generate ZIP of multiple copies + answer key
- `POST /api/exams/:id/pdf-zip`
- Body:
```json
{
  "copies": 30,
  "institution": "Acme School"
}
```
- Response: `application/zip`

Answer key CSV
- `POST /api/exams/:id/answer-key`
- Body: `{ "variant": <ExamVariant> }`
- Response: `text/csv`

Corrections
- `POST /api/corrections`
- Content-Type: `multipart/form-data`
- Fields:
- `keyCsv` (file)
- `answersCsv` (file)
- `mode` (optional, `strict` or `lenient`, defaults to `strict`)

Response
```json
{ "results": [{ "examNumber": "A001", "studentId": "123", "score": 8, "details": [] }] }
```

**CSV Formats**
Answer key CSV (generated by the app)
```
exam-id,question1-answer,question2-answer,question3-answer...
```

Student answers CSV (uploaded for correction)
```
exam-id,cpf,question1-answer,question2-answer,question3-answer...
```

Rules
- Use `|` (pipe) to separate multiple selections in letters mode.
- In powers-of-two mode, each answer can be a single number (sum) or multiple values separated by `|` or `,`.
- The corrections endpoint accepts either comma or semicolon delimiters.
